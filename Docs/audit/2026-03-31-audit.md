| C1 | `Sources/Anthropic/Client/RequestPipeline.swift` | 39–43, 65–74 | software-engineer, qa-engineer | `RequestPipeline` casts `httpClient as? URLSessionHTTPClient` to call the internal `send(urlRequest:)` / `stream(urlRequest:)` methods, bypassing the `HTTPClient` protocol. This violates the CLAUDE.md rule "All HTTP calls via `HTTPClient` protocol" and breaks testability for any custom `HTTPClient` that is not `URLSessionHTTPClient`. | Restructure so `URLSessionHTTPClient` accepts a pre-built `URLRequest` through its `HTTPClient.send(_:HTTPRequest)` conformance (the `HTTPRequest` already carries everything needed). Remove the internal bypasses and the `fatalError` stubs. |
| C2 | `Sources/Anthropic/Networking/URLSessionHTTPClient.swift` | 18, 82 | security-engineer, software-engineer | `URLSessionHTTPClient.send(_:HTTPRequest)` and `URLSessionHTTPClient.stream(_:HTTPRequest)` call `fatalError(...)`. These are public protocol conformances; any conforming type that is accidentally used via the `HTTPClient` protocol will crash the host app in production with no recovery path. | Implement the `HTTPClient` protocol methods properly so the pipeline can call them without the internal cast bypass. |
| C3 | `Sources/Anthropic/Client/RequestPipeline.swift` | 66–70 | qa-engineer | `RequestPipeline.stream()` uses `try?` to build the `URLRequest`, silently losing the error type. The resulting stream that immediately throws an `AnthropicError.encodingError` is never exercised by any test. | Add a service-level test for the encoding-failure path in `stream()`. |
